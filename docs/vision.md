# ghost-archives: 프로젝트 비전

## 전제

'아카이브'라는 단어는 하나의 기표 아래 전혀 다른 기의들이 공존한다. 패션 브랜드의 "아카이브 컬렉션"과 공공기관의 "기록 아카이브"는 같은 단어를 쓰지만 가리키는 세계가 다르다. 이 프로젝트는 그 분기를 추적한다.

---

## 질문

1. '아카이브'라는 단어는 지금 어디서, 어떤 의미로 소비되고 있는가?
2. 이 단어의 의미가 여러 갈래로 나뉘게 된 이유는 무엇인가?
3. 시간이 지나면서 각 의미의 비중은 어떻게 변하는가?

---

## 현재 할 수 있는 것 (Phase 1-2 완료 기준)

- 뉴스 기사 자동 수집 (Google News RSS + 네이버)
- 제목 기반 7개 카테고리 자동 분류 (Gemini)
- 카테고리별 수량 분포 파악
- 주간 수집 추이 시각화

→ "예술/제도 용례가 압도적이고, 기술 용례는 거의 없다"는 정도의 사실 확인.

---

## 아직 할 수 없는 것 — 그러나 해야 하는 것

### A. 이벤트 클러스터링 (데이터 정규화)

**문제:** 동일 이벤트에 대해 여러 매체가 기사를 쏟으면 개수가 뻥튀기된다. "예술 234건"이 234개의 서로 다른 용례인지, 10개 이벤트의 반복 보도인지 구분 불가.

**목표:** 기사 단위 → 이벤트 단위로 정규화. "고유 이벤트 수"와 "기사 수"를 분리.

**구현 계획:**

1. **유사도 기반 그룹핑 스크립트** (`scripts/cluster.py`)
   - 같은 카테고리 내에서, 발행일 ±3일 이내, 제목 유사도 70% 이상 → 같은 이벤트로 묶기
   - 유사도: 간단한 토큰 자카드 유사도 or Gemini에게 "이 두 제목은 같은 사건인가?" 판별
   - 결과: 각 아이템에 `event_id` 필드 부여

2. **데이터 스키마 확장**
   - `event_id`: 같은 이벤트 그룹 ID (예: `evt-2026-04-001`)
   - `event_representative`: boolean — 이벤트 내 대표 기사 여부

3. **대시보드 반영**
   - 카테고리 분포를 "기사 수" vs "이벤트 수" 토글로 표시
   - 이벤트 수 기준 분포가 더 정직한 비율

4. **목록 페이지 반영**
   - 같은 이벤트 기사는 접어서 표시 (대표 1건 + "외 N건")

**난이도:** 중. 제목 유사도 계산은 단순 구현 가능. Gemini 호출 없이도 1차 클러스터링 가능.

---

### B. implied_meaning 구조화 (카테고리 내부 의미 세분화)

**문제:** 현재 `implied_meaning`은 "10자 이내 요약"으로 자유 텍스트. 같은 의미도 다르게 표현되어 집계 불가. 예: "전시 형식", "전시로서", "전시 방법" → 같은 뜻이지만 3개로 카운트.

**목표:** 1뎁스 카테고리(7개)는 유지. 각 카테고리 내부에서 의미의 분기를 질문 형태로 추적.

**1차 분석 결과 (2026-05-07):**

#### 예술 (234건) — 가장 이질적
- "게임 콘텐츠 모음"이 33건으로 최다. 게임이 예술인가, 마케팅인가, 큐레이션인가?
- 내부 스펙트럼: 기록(춤/공연) vs 소장(미술관) vs 모음(작품집) vs 게임
- 질문: "예술 아카이브"에서 '아카이브'는 형식인가 재료인가 기관인가?

#### 제도 (217건) — 가장 균질
- "공공 기록 보존/보관소/보관" 계열이 82건(38%). 의미 분산 적음
- 질문: 제도 아카이브는 하나의 의미로 수렴하고 있는가? 그렇다면 왜?

#### 마케팅 (88건) — 헤리티지 서사
- "과거 컬렉션" "브랜드 역사" "브랜드 헤리티지" → 과거를 자산화하는 전략
- 게임 콘텐츠가 여기에도 등장 (아이템 묶음, 서비스 기록)
- 질문: 마케팅에서 "아카이브"는 왜 "과거"와 결합하는가? 복각/재발매와의 관계

#### 감성 (53건) — 노스탤지어 수렴
- 거의 전부 "과거의 기록/추억" 계열. 다양성 가장 낮을 가능성
- 질문: 감성 아카이브 = 노스탤지어 아카이브인가? 그 외 감성적 용법은 있는가?

#### 큐레이션 (47건) — "모음" 일색
- "과거 기사/영상/사진/뉴스 모음"
- 질문: 큐레이션과 아카이브의 차이는 무엇인가? 이 카테고리의 존재 이유는?

#### 시민 (47건) — 가장 정치적
- "기록 보존 및 공유" "사회운동 기록" "지역 역사"
- 질문: 시민 아카이브에서 "기록"은 보존인가 저항인가?

#### 기술 (34건) — 가장 리터럴
- "데이터 저장소" "게임 데이터 보관소" "압축 파일"
- 질문: 기술적 의미의 archive가 왜 한국어 뉴스에서는 드문가?

**경계 문제: 게임**
- 게임 관련 기사가 예술/마케팅/기술/큐레이션에 분산. 모델이 일관성 없이 분류.
- 선택지: ① 게임을 하나의 카테고리에 강제 ② 게임 자체가 카테고리 경계를 드러내는 사례로 남겨두기
- 후자가 더 흥미로운 발견일 수 있음 — "게임 아카이브"는 여러 의미를 동시에 품는 사례

**다음 단계:**

1. **기존 implied_meaning 분석** (`scripts/analyze_meanings.py`)
   - 카테고리별로 implied_meaning 값 빈도 집계
   - 유사 표현 그룹핑 (수동 검토 + Gemini 보조)
   - 각 카테고리에 3~5개 하위 의미 유형(sub_meaning) 도출

2. **하위 의미 체계 정의** (예시)
   - 예술: `전시 형식` / `창작 재료` / `보존 행위` / `아카이빙 작품` / `기관 컬렉션`
   - 마케팅: `헤리티지 브랜딩` / `빈티지 상품` / `컬렉션 명칭` / `복각·재발매`
   - (데이터 분석 후 귀납적으로 확정)

3. **분류 스크립트 업데이트** (`classify.py`)
   - 프롬프트에 `sub_meaning` 필드 추가
   - 카테고리별 하위 유형 선택지를 명시
   - 기존 분류 항목도 재분류 (일괄 배치)

4. **데이터 스키마 확장**
   - `sub_meaning`: 카테고리 내 하위 의미 유형 (정규화된 태그)
   - `implied_meaning`: 유지 (자유 텍스트, 보조 참고용)

5. **시각화**
   - 카테고리 클릭 시 하위 의미별 분포 (서브 차트)
   - 목록 페이지에서 sub_meaning으로 2차 필터링

**난이도:** 중~상. 하위 의미 체계를 정의하는 과정이 핵심이고 이건 사람(담)의 판단 필요. 구현 자체는 classify.py 프롬프트 수정 수준.

---

### C. 카테고리별 타임라인 (서사 추적)

**문제:** 시간 추이 차트는 "전체 수집량"만 보여준다. 카테고리별로 언제 어떤 이벤트가 있었는지, 특정 의미가 급증한 시점이 있는지 추적 불가.

**목표:** 카테고리별 시계열을 분리하고, 주요 이벤트를 타임라인에 마킹.

**구현 계획:**

1. **대시보드 확장** (`dashboard.html`)
   - 카테고리별 수집 추이 (스택 차트 또는 카테고리 선택식 라인 차트)
   - 월별 카테고리 비율 변화 (100% 스택 바)

2. **이벤트 마킹** (A 완료 후)
   - 이벤트 클러스터 중 기사 5건 이상인 것을 "주요 이벤트"로 표시
   - 타임라인에 점 + 라벨로 표기
   - 예: 2026-04-15 "국립현대미술관 아카이브전 개막" (예술, 12건)

3. **스냅샷 비교**
   - 월말마다 카테고리 비율 스냅샷 저장 (`data/snapshots/`)
   - "지난 달 대비 감성 +3%, 제도 -2%" 식 변화량 표시

4. **분류 기준 변경 이력** (`docs/classification-log.md`)
   - 카테고리 추가/병합/분리 시 기록
   - 이전 기준과 현재 기준의 비교 가능성 유지

**난이도:** 중. Chart.js로 스택 차트 구현은 단순. 스냅샷 저장은 스크립트 하나 추가. 이벤트 마킹은 A 완료 전제.

---

### 의존 관계와 순서

```
A (이벤트 클러스터링)
  → C의 이벤트 마킹에 필요
  → 독립 실행 가능

B (implied_meaning 구조화)
  → 기존 데이터 분석부터 시작 (스크립트)
  → 하위 체계 정의는 사람 판단 필요
  → 독립 실행 가능

C (카테고리 타임라인)
  → 기본 차트는 바로 가능
  → 이벤트 마킹은 A 완료 후
  → 스냅샷은 독립
```

**권장 순서:**
1. B-1 (기존 implied_meaning 분석) → 현재 데이터 품질 파악
2. A (이벤트 클러스터링) → 데이터 신뢰도 확보
3. B-2~5 (하위 의미 체계 확정 + 재분류)
4. C (타임라인 + 스냅샷)

## 이 프로젝트의 최종 산출물

1. **데이터셋** — '아카이브' 용례 아카이브 (메타적 구조)
2. **시각화** — 의미의 분포와 변화를 보여주는 대시보드
3. **글** — 〈아카이브라는 이름의 유령〉: 데이터를 근거로 한 비평적 에세이

데이터셋과 시각화는 글의 논거를 구축하기 위한 도구이자, 그 자체로 독립적인 결과물이다.

## 태도

- 수집은 증거 확보다
- 분류는 해석의 시작이다 (끝이 아니다)
- 분류 체계는 고정된 것이 아니라 데이터와 함께 진화한다
- '아카이브'라는 이름 자체가 분석 대상이다
- 양적 분포는 질적 해석을 위한 출발점이다
