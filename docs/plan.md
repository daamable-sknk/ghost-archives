# ghost-archives 작업 계획

> **현재 초점: 웹 (Track D)** — 사이트를 탐색·관찰 가능한 아카이브로 만드는 것.  
> 글(Track F)은 웹이 충분히 돌아간 뒤, 데이터를 글로 옮기는 단계.

> **출처:** 강좌 「아키비스트는 웹을 꿈꾸는가」  
> **01-4** 웹에 기반한 아카이브

ghost-archives는 01-4의 **실습·연구 프로젝트**다.  
강의에서 제시한 「아카이브 = 기록 + 공간 + 의미」와 「아카이브라는 이름의 유령(기표/기의 분기)」를, **웹 위에서 직접 만들고 관찰**하는 것이 목표다.

---

## 01-4와 프로젝트의 대응 관계

| 01-4 개념 | ghost-archives에서의 구현 | 상태 |
|---|---|---|
| **기록** — 더 이상 쪼개면 의미를 잃는 최소 단위 | 뉴스·SNS 등에서 잡힌 용례 1건 = `ghosts.json` 항목 (제목·URL) | ✓ 수집 중 |
| **공간** — 기록이 질서를 갖는 장소 | GitHub Pages 정적 사이트 (`ghost.meta-archives.xyz`) | ✓ 운영 중 |
| **의미** — 무엇이 말해지고 말해지지 않는가를 결정하는 질서 | 7개 카테고리 + `implied_meaning` 분류 | ✓ 자동화 |
| **유령** — 같은 기표, 다른 기의 | 카테고리별 분포·하위 의미 분석 | △ 데이터만, 해석 미완 |
| **의미의 스펙트럼** — 9개 분야 표 | 7개 카테고리 (강의 표와 근사 매핑, 아래 참고) | △ 뉴스 편중 |
| **스펙트럼 형성 배경** — 5가지 이유 | 시계열·소스 확장으로 검증 예정 | ✗ |
| **웹이 문턱을 낮춤** — 생산자 확대 | 뉴스 외 SNS·브랜드·Are.na 수집 | ✗ |
| **웹의 시간성** — 링크 부패, 영속 | URL 유효성·수집 기간 표시 | △ 기간만 |
| **핸드메이드 웹** — 작고 가볍고 독립적 | 순수 HTML/CSS/JS, JSON 데이터 | ✓ |
| **디지털 가드닝** — 통제 가능한 나의 웹 | 개인 도메인 + Git 레포 | ✓ |
| **글** — 〈아카이브라는 이름의 유령〉 | meta-archives.xyz 본문 | ✗ 예정 |

### Track F. 글쓰기 — 〈아카이브라는 이름의 유령〉

> 01-4 마지막: *「아키비스트도 웹을 꿈꿀 수 있을까. 직접 웹으로 아카이브를 만드는 경험이 필요하지 않을까.」*

**목표:** ghost-archives 데이터 + 사이트가 meta-archives.xyz 글의 **근거·실례·부록**.

| 글의 장 | ghost-archives에서 가져갈 것 |
|---|---|
| 유령 — 기표/기의 분기 | 카테고리 분포 차트, 대표 용례 21건 |

| 핸드메이드 웹 | 이 사이트 자체 — 구조·코드·지속성 |
| 영속하는 아카이브 | Git + JSON + GitHub Pages 전략 |
